The recipe feature is still in-development at this time.  This will outline outstanding items for the V0 of the feature as well as V1 features, and applications that will use the recipe feature.

Requirements


Recipe V0

V0 of the recipe feature should include fundamental components of the feature so it is usable and valuable, knowing there are opportunities to expand upon the feature in the future.


Requirements

Save: Users should be able to save recipes and the save process should analyze all historical data on initial save.

General Cleanup: We need to allocate some time for general cleanup including picker components, constraint grouping, and table display.

Cookbook Selection: To ensure we are able to retrieve the correct recipe for users, we need to help operators select the appropriate cookbook within the Operator Centerline.  We need to do this for the Process Engineer Centerline application as well.


Recipe V0.1


Requirements

Automated Update: Saved recipes should automatically check for new “golden runs” daily and update the recipe as needed.

Automated Categorical Grouping: For new categorical values as part of the constraint grouping, recipe groups should automatically get created and updated.


Recipe V1

Once the recipe feature is though V0, there are a number of improvements that should be added.


Requirements

KPI Outcome: KPIs should be integrated into the feature as outcomes available for selection

Template: Creating each individual recipe group is time consuming.  Users should be able to create a template of constraints, outcomes, filters, and levers that apply to all recipe groups by default but can be modified.

Run-level Data: Allow users to select runs in the table and display run-level data.

Visualization: Allow users to chart data fields in the table within a chart. Users should be able to select a y1, y2, and x-axis.


Centerline Comparison


Process Engineer

The centerline comparison tool allows users to determine what the largest differences are between a time in which they were operating optimally (“golden run”) and when they are running poorly.  This application has been developed as a custom application and needs to be made available off the shelf.


Requirements

Add modes for:

Manual: Allow users to find the centerline range via the sidebar

Automatic: Find the centerline by retrieving the calculated recipe based on the analysis time period

If multiple recipes are needed, warn the user and have them select one at a time

Related tickets: SU-812:  Centerline Comparison tool needs manual entry option for LCL and UCLBACKLOG


Operator

For operators, it is important to quickly understand the difference between current set points (levers) and ideal set points (recipes).  Operators are also often provided more information than they need while they’re trying to resolve issues and change operating set points to accommodate ever changing. conditions. For operators, we’d like to simplify the centerline comparison.


Requirements

Automatically retrieving the appropriate recipe based on current conditions

Operators should be able to change the selected recipe if needed

Operators should be presented with additional information, like current conditions

This aids in trust of the recipe

Future: As process engineers propose new recipes, those should be available to operators through this applicaiton
